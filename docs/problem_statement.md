# Постановка задачи

## Актуальность
В своей курсовой работе я разрабатываю прототип интерфейса B2B-маркетплейса для компаний, которые закупают товары оптом.

Мне важно показать интерфейс, где пользователь сможет быстро найти нужный товар, посмотреть информацию о поставщике, сравнить предложения и отправить заявку на закупку.

## Ограничения
Я не делаю полноценный сайт с базой данных и серверной частью. В работе будет представлен дизайн, макет, описание экранов и пользовательская логика.

## Критерии успешности
1. Интерфейс должен быть понятным для компании-покупателя.
2. В макете должны быть каталог, карточка товара, информация о поставщике и заявка.
3. Структура должна подходить именно для оптовых закупок.
4. Проект должен быть понятен преподавателю без дополнительных пояснений.

## Риски
- Интерфейс может получиться слишком похожим на обычный интернет-магазин.
- Макет может быть перегружен лишними элементами.
- Может не хватить анализа похожих B2B-платформ.
